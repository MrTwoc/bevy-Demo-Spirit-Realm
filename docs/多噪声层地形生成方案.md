# 多噪声层地形生成方案

> **目标**：从当前单调的单层 FBM 丘陵，升级为具备海洋、河流、峡谷、山脉、洞穴等多样化地形的 Minecraft 级生成系统。
>
> **适配垂直世界范围**：Y 轴 ±10240（自然地形全范围）/ ±20480（建筑）
>
> **当前状态**：单层 [`Fbm<Simplex>`](../src/chunk.rs:544-551) 2D 高度图，Octaves=4, Freq=0.005, 仅产生缓坡丘陵

---

## 1. 现状分析

### 1.1 当前实现

现有地形生成位于 [`src/chunk.rs:560`](../src/chunk.rs:560) 的 [`fill_terrain()`](../src/chunk.rs:560) 函数：

```rust
// 当前单一噪声层
const TERRAIN_BASE_HEIGHT: i32 = 16;   // 基准高度
const TERRAIN_AMPLITUDE: f64 = 32.0;   // 起伏幅度
const TERRAIN_SEED: u32 = 42;

fn create_terrain_noise() -> Fbm<Simplex> {
    Fbm::<Simplex>::new(TERRAIN_SEED)
        .set_octaves(4)
        .set_frequency(0.005)
        .set_lacunarity(2.0)
        .set_persistence(0.5)
}
```

### 1.2 局限性

| 问题 | 表现 | 原因 |
|------|------|------|
| 地形单调 | 只有缓坡丘陵，无山脉/峡谷/海洋 | 单层 2D 噪声无法表达多样地形 |
| 无海洋 | 世界全是陆地，无水域 | 噪声值映射到高度，未分离大陆/海洋 |
| 无洞穴 | 地下全是实心方块 | 无 3D 噪声生成洞穴结构 |
| 无河流 | 地表无河谷切割 | 缺少侵蚀噪声层 |
| 无生物群系 | 全地形同质 | 无温度/湿度噪声 |

## 2. 多噪声层架构总览

### 2.1 核心思想

借鉴 Minecraft 的 **MultiNoise** 地形生成方法，将多个独立的噪声层叠加混合，每层负责一种地形特征：

```
输入：区块坐标 (cx, cy, cz)
    │
    ├── [2D] 大陆性噪声  ──→ 大陆/海洋 二元分类
    ├── [2D] 地势噪声     ──→ 平原/高原/山脉 高度骨架
    ├── [2D] 侵蚀噪声     ──→ 地形粗糙度（峡谷/河谷）
    ├── [2D] 山峰噪声     ──→ 尖锐山峰 vs 圆润丘陵
    ├── [2D] 温度噪声     ──→ 生物群系温度带
    ├── [2D] 湿度噪声     ──→ 生物群系湿度带
    └── [3D] 洞穴噪声     ──→ 地下空洞与隧道
    │
    ▼
地形混合器 ──→ 最终方块填充
```

### 2.2 噪声层分解

| 噪声层 | 维度 | 频段 | 用途 | 输出范围 |
|--------|------|------|------|---------|
| **大陆性** | 2D (XZ) | 极低频 (~0.001) | 决定大陆/海洋 | [-1.0, 1.0] |
| **地势** | 2D (XZ) | 低频 (~0.003) | 地形高度骨架 | [-1.0, 1.0] |
| **侵蚀** | 2D (XZ) | 中频 (~0.008) | 峡谷/河谷切割 | [0.0, 1.0] |
| **山峰** | 2D (XZ) | 中高频 (~0.02) | 山峰尖锐度 | [-1.0, 1.0] |
| **温度** | 2D (XZ) | 超低频 (~0.0005) | 纬度温度带 | [-1.0, 1.0] |
| **湿度** | 2D (XZ) | 低频 (~0.002) | 干燥/湿润 | [-1.0, 1.0] |
| **洞穴** | 3D (XYZ) | 中高频 (~0.03) | 地下空洞 | [-1.0, 1.0] |

### 2.3 噪声配置表

所有噪声层基于同一根种子 `BASE_SEED`，每层使用不同的子种子以保持独立性：

```rust
const BASE_SEED: u32 = 42;

// 各噪声层使用不同的子种子偏移
const CONTINENTAL_SEED_OFFSET: u32 = 0;

const EROSION_SEED_OFFSET: u32     = 2000;
const PEAK_SEED_OFFSET: u32        = 3000;
const TEMPERATURE_SEED_OFFSET: u32 = 4000;
const HUMIDITY_SEED_OFFSET: u32    = 5000;
const CAVE_SEED_OFFSET: u32        = 6000;
```

| 参数 | 大陆性 | 地势 | 侵蚀 | 山峰 | 温度 | 湿度 | 洞穴 |
|------|--------|------|------|------|------|------|------|
| Octaves | 2 | 4 | 3 | 3 | 2 | 3 | 3 |
| Frequency | 0.001 | 0.003 | 0.008 | 0.02 | 0.0005 | 0.002 | 0.03 |
| Lacunarity | 2.0 | 2.0 | 2.0 | 2.5 | 2.0 | 2.0 | 2.0 |
| Persistence | 0.5 | 0.5 | 0.5 | 0.3 | 0.5 | 0.5 | 0.5 |

## 3. 五大基本地形生成

### 3.1 地形分类

```
五大基本地形：
┌─────────────────────────────────────────────────────────────┐
│                     地形分类树                              │
│                                                             │
│  大陆性噪声 > 0  ───→ 大陆                                  │
│      ├── 地势高 + 山峰高  ──→ 山脉                          │
│      ├── 地势高 + 山峰低  ──→ 高原                          │
│      ├── 地势中 + 侵蚀低  ──→ 平原                          │
│      ├── 地势中 + 侵蚀高  ──→ 丘陵                          │
│      └── 地势低            ──→ 盆地                          │
│                                                             │
│  大陆性噪声 ≤ 0 ───→ 海洋                                  │
│      ├── 深度浅  ──→ 浅海/大陆架                            │
│      ├── 深度中  ──→ 深海                                  │
│      └── 深度大  ──→ 海沟                                  │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 大陆与海洋

**大陆性噪声** 是最顶层的分类器。它是一个超低频 2D Simplex 噪声，输出平滑的大尺度结构：

```
大陆性噪声值:
    > 0.2   → 大陆中心（地形完整）
    0.0~0.2 → 海岸带（海滩/悬崖过渡）
    < 0.0   → 海洋
```

**海洋深度计算**：

```rust
fn get_ocean_depth(coord: &ChunkCoord, x: usize, z: usize) -> i32 {
    let continental = sample_continental(coord, x, z);
    if continental > 0.0 {
        return 0; // 陆地
    }
    // 海洋深度：噪声值越负，海水越深
    // 范围：-1.0 ~ 0.0 → 深度 0 ~ -10240 格（适配全范围）
    (continental * 10240.0) as i32
}
```

### 3.3 地势噪声 → 高度骨架

**地势噪声** 是地形的"骨架"，决定了地形的宏观高低分布。它与 **大陆性噪声** 结合得到基础高度：

```rust
fn get_base_height(continental: f64, elevation: f64) -> f64 {
    if continental > 0.0 {
        // 大陆区域：地势噪声映射到高度
        // 幅度放大至 10240 以覆盖 ±10240 全范围
        elevation * TERRAIN_AMPLITUDE
    } else {
        // 海洋区域：高度为负（海床），最深可达 -10240
        continental * TERRAIN_AMPLITUDE  // 海床最深 -10240
    }
}
```

### 3.4 峰谷噪声 → 地形细节

**山峰噪声** 用于区分三种地形形态：

| 山峰值 | 形态 | 典型地形 |
|--------|------|---------|
| > 0.3 | 尖锐山峰 | 山脉/峭壁 |
| -0.3 ~ 0.3 | 圆润起伏 | 丘陵/缓坡 |
| < -0.3 | 平坦 | 平原/高原 |

```rust
fn apply_peak_modifier(base_height: f64, peak: f64, erosion: f64) -> f64 {
    if peak > 0.3 {
        // 尖锐山峰：二次方放缩，拉高顶部
        let peak_factor = (peak - 0.3) / 0.7; // 归一化到 [0, 1]
        base_height + peak_factor.powi(2) * 5120.0  // 山峰加成 ~5120 格
    } else if peak < -0.3 {
        // 平坦区域：压低起伏
        let flat_factor = (-0.3 - peak) / 0.7; // 归一化到 [0, 1]
        base_height * (1.0 - flat_factor * 0.5)
    } else {
        // 中间区域：保持原样
        base_height
    }
}
```

### 3.5 侵蚀噪声 → 地形切割

**侵蚀噪声** 模拟水流对地形的切割作用，形成峡谷、河谷等地貌：

```rust
fn apply_erosion(height: f64, erosion: f64, elevation: f64) -> f64 {
    // 侵蚀只在坡度足够大的区域产生效果
    let slope = /* 计算坡度 */;
    
    // 侵蚀量 = 侵蚀噪声 × 坡度因子 × 高度因子
    let erosion_amount = erosion * slope * (elevation.max(0.0) / 1.0);
    
    height - erosion_amount * 1024.0  // 侵蚀幅度放大至 1024
}
```

侵蚀效果汇总：

| 侵蚀值 | 地形特征 | 视觉效果 |
|--------|---------|---------|
| > 0.6 | 强侵蚀 | 深邃峡谷、陡峭河谷 |
| 0.3~0.6 | 中侵蚀 | V 形山谷、起伏丘陵 |
| 0.0~0.3 | 弱侵蚀 | 缓坡、U 形谷 |
| < 0.0 | 沉积 | 冲积平原、三角洲 |

### 3.6 五大地形生成算法汇总

```rust
pub enum TerrainType {
    Plains,     // 平原
    Hills,      // 丘陵
    Plateau,    // 高原
    Mountains,  // 山脉
    Basin,      // 盆地
    Ocean,      // 海洋
}

pub fn classify_terrain(
    continental: f64,
    elevation: f64,
    peak: f64,
    erosion: f64,
) -> TerrainType {
    if continental <= 0.0 {
        return TerrainType::Ocean;
    }

    match (elevation, peak, erosion) {
        // 山脉：高海拔 + 高峰值
        (e, p, _) if e > 0.4 && p > 0.2 => TerrainType::Mountains,
        // 高原：高海拔 + 低侵蚀 + 低峰值
        (e, _, er) if e > 0.4 && er < 0.2 => TerrainType::Plateau,
        // 盆地：低海拔 + 周围高（需检测局部对比度）
        (e, _, _) if e < -0.2 => TerrainType::Basin,
        // 丘陵：中等侵蚀
        (_, _, er) if er > 0.3 => TerrainType::Hills,
        // 平原：剩余情况
        _ => TerrainType::Plains,
    }
}
```

## 4. 特色地形特征生成

### 4.1 河流系统

河流是贯穿多种地形的线性特征。通过以下方式生成：

1. **河流路径**：使用低频噪声生成河流的"源点"场，追踪梯度下降路径
2. **河谷挖掘**：沿河流路径降低地表高度
3. **水体填充**：河谷底部填充水源方块

**简化方案**（Phase 1 实现）：

```rust
fn apply_rivers(
    height_map: &mut [[f64; 32]; 32],
    coord: &ChunkCoord,
    elevation_noise: &Fbm<Simplex>,
) {
    // 使用叠加低频噪声检测"河流可能性"
    let river_noise = /* 专用的河流路径噪声 */;
    
    for z in 0..32 {
        for x in 0..32 {
            if river_noise.get([world_x, world_z]) > RIVER_THRESHOLD {
                // 沿河流中心线加深地形
                let depth = (river_noise - RIVER_THRESHOLD) * RIVER_DEPTH;
                height_map[z][x] -= depth;
            }
        }
    }
}
```

### 4.2 洞穴系统

洞穴使用 **3D Simplex 噪声** 生成。与 2D 噪声不同，3D 噪声在 XYZ 三个维度采样，可以产生真正的三维空洞结构。

**核心算法**：

```rust
fn is_cave_block(world_x: i32, world_y: i32, world_z: i32) -> bool {
    // Y 轴只在地下层（地表以下）生成洞穴
    let surface_y = get_surface_height(world_x, world_z);
    if world_y >= surface_y - 2 {
        return false; // 地表附近不生成洞穴
    }
    if world_y < surface_y - 128 {
        return false; // 过深区域也不生成（节省计算）
    }
    
    // 3D 噪声采样
    let cave_noise = get_cave_noise_3d(world_x, world_y, world_z);
    
    // 用多个 octave 叠加产生洞穴结构
    // 正值 → 实心，负值 → 空洞
    cave_noise > CAVE_THRESHOLD
}
```

**洞穴类型**：

| 类型 | 噪声频率 | 阈值 | 特征 |
|------|---------|------|------|
| 隧道洞穴 | 0.03 | 0.2 | 长条通道，适合探索 |
| 大厅洞穴 | 0.02 | 0.1 | 大空间穹顶 |
| 海绵洞穴 | 0.05 | 0.3 | 小孔洞群，类似瑞士奶酪 |
| 裂缝 | 0.04 + 各向异性 | 0.0 | 狭窄垂直裂缝 |

## 5. 垂直地层结构

### 5.1 标准地层剖面

Y 轴从下到上（±10240 全范围自然地形，基岩在 -20480）：

```
Y 高度         方块类型        说明
─────────────────────────────────────────────────
+10240        空气 / 雪峰     最高自然地表可达（山峰）
  │
  ┌ 地表高度                   由噪声决定（±10240 范围内）
  ├──────────────────────────── 海拔分层线
  │  Y > 512:  石头/雪         高海拔裸露基岩，无表土
  │  Y 0~512:  草方块/泥土/石  标准海拔表土层
  │  Y -64~0:  沙子/石头       低海拔/海岸带
  │  Y -512~-64: 石头          浅层岩石
  ├ 深板岩 (-5120 ~ -512)      深层岩石 ~4608 格
  ├ 玄武岩/地幔 (-10240~-5120) 地幔过渡带 ~5120 格
  ├ 基岩 (-20480 ~ -10240)     致密基岩 ~10240 格
  └ 虚空 (< -20480)            世界边界 / 最低生成下限
```

### 5.2 垂直分层填充算法（纯绝对 Y 值分层）

```rust
fn get_block_for_height(
    world_y: i32,
    surface_height: i32,
) -> u16 {
    if world_y > surface_height {
        return 0; // 空气
    }
    
    // 地表方块：由绝对 Y 值决定海拔层
    if world_y == surface_height {
        return get_surface_block(world_y);
    }
    
    // 地下方块：纯绝对 Y 值分层，不再使用 below_surface 相对深度
    get_subterranean_block(world_y, surface_height)
}

/// 地表方块类型由绝对海拔决定
fn get_surface_block(y: i32) -> u16 {
    match y {
        y if y >= 1280  => 9,  // 雪（高山永久积雪带）
        y if y >= 512   => 2,  // 石头（高山裸露，无表土）
        y if y >= 8     => 1,  // 草方块（正常海拔）
        y if y >= -64   => 4,  // 沙子（低海拔/海岸带）
        y if y >= -512  => 2,  // 石头（深海床）
        _               => 6,  // 深板岩（极深海床/深渊）
    }
}

/// 地下方块：按绝对 Y 值分层
fn get_subterranean_block(y: i32, surface_height: i32) -> u16 {
    // 标准海拔（-64 ~ +512）：保留薄层表土改善手感
    if y >= -64 && y <= 512 {
        let below_surface = surface_height - y;
        if below_surface <= 4 {
            return 3;  // 泥土（仅标准海拔有表土）
        }
    }
    
    // 纯绝对 Y 值分层
    match y {
        y if y >= 512   => 2,  // 石头（高海拔地下）
        y if y >= -512  => 2,  // 石头（中浅层岩石）
        y if y >= -5120 => 6,  // 深板岩（深层岩石 ~4608 格厚）
        y if y >= -10240 => 8, // 玄武岩/地幔岩（地幔过渡带）
        y if y >= -20480 => 7, // 基岩（致密基岩层 ~10240 格厚）
        _               => 0,  // 虚空（低于 -20480）
    }
}
```

### 5.3 ±10240 全范围垂直分层

由于自然地形覆盖整个 ±10240 范围，按绝对 Y 值划分为以下地层带：

| Y 范围 | 地层 | 说明 |
|--------|------|------|
| +512 ~ +10240 | 高海拔岩石/雪原 | 地表裸露基岩或雪盖，**无表土**，树木无法生长 |
| -64 ~ +512 | **标准表层带** | 草方块/泥土/石头，主要游戏区域 |
| -512 ~ -64 | 低海拔岩石/海床 | 浅层岩石或沙质海床 |
| -5120 ~ -512 | 深板岩带 | 深层岩石，可分布珍稀矿石 |
| -10240 ~ -5120 | 玄武岩/地幔带 | 地幔过渡区，岩浆池分布 |
| -20480 ~ -10240 | **基岩层** | 致密基岩，不可挖掘 |
| < -20480 | 虚空 | 世界边界 |

> 得益于 **实时流式加载**（`Y_LOAD_RADIUS=4`，约 ±128 格），每帧只需加载玩家附近约 8 层 Y 的区块，与原来的加载量一致。远 Y 层的区块无需预生成，由地表高度预计算（O(1)）快速决定其地层类型，返回 [`ChunkData::Uniform`](../src/chunk.rs:262) 而非 [`ChunkData::Empty`](../src/chunk.rs:262)。

### 6.2 数据流

```
    │
    ▼

    │
    ▼
[terrain::fill_terrain_multi_noise()]
    ├── 步骤 1: 采样所有 2D 噪声层（大陆/地势/侵蚀/山峰/温湿度）

    ├── 步骤 3: 计算地表高度（合并各噪声层）
    ├── 步骤 4: 填充 32³ 体素（地层垂向分层）
    ├── 步骤 5: 3D 洞穴剔除（如果 cy 在地下范围）
    ├── 步骤 6: 河流/峡谷调整
    └── 步骤 7: 返回 ChunkData
    │
    ▼
[AsyncMeshManager] 异步生成网格
```

## 7. 性能分析与优化

### 7.1 计算成本估算

| 操作 | 单区块耗时估计 | 说明 |
|------|-------------|------|
| 6 个 2D 噪声层采样 | ~0.03ms | 每区块 32×32=1024 次，每次约 6 个采样 |
| 地形分类与高度计算 | ~0.01ms | 纯数学运算 |
| 垂向填充 32³ | ~0.05ms | 32768 次循环 |
| 3D 洞穴剔除 | ~0.1ms | 仅地下区块需要 |
| **总计** | **~0.2ms** | 比当前单层噪声 (~0.01ms) 慢 20 倍 |

> **注意**：±10240 全范围地形并不会增加每帧加载量。由于系统采用 [**实时流式加载**](../src/chunk_manager.rs:191)（`Y_LOAD_RADIUS=4`，每帧加载 ~8 层 Y ≈ 2,312 区块），与修改前一致。更深/更高的区块按需惰性加载，不会一次性全部生成。

### 7.2 优化策略

| 优化 | 预期收益 | 实现难度 |
|------|---------|---------|
| **地表高度预计算**：每 XZ 列先用 2D 噪声算地表高度，O(1) 决定每 chunk 策略（Empty/Uniform/Mixed） | 深部 ~95% 区块跳过详细生成 | 低 |
| **共用噪声采样**：每列 XZ 点的噪声值在 Y 循环中缓存 | 减少 80% 重复采样 | 低 |
| **深部 Uniform 跳过网格**：`ChunkData::Uniform` 直接跳过 MeshTask | 网格生成减少 ~95% | 低 |
| **预计算地形骨架**：将 2D 噪声结果缓存在 MegaColumn 级 | 按列增量加载 | 中 |
| **GPU 噪声迁移**：将噪声计算迁移到 Compute Shader | 释放 CPU，并行化 | 高（远期） |

### 7.3 量化对比

| 指标 | 当前（单层 FBM） | 优化后（多噪声层） |

## 10. GPU 计算可行性分析

### 10.1 各噪声层的 GPU 适配性

| 噪声层 | 维度 | GPU 适配度 | 理由 |
|--------|------|-----------|------|
| **大陆性** | 2D (XZ) | ⭐⭐⭐⭐⭐ **极易** | 纯 2D 噪声，每个体素独立，完美适配 Compute Shader |
| **地势** | 2D (XZ) | ⭐⭐⭐⭐⭐ **极易** | 同上，与大陆性噪声可合并到同一 dispatch |
| **侵蚀** | 2D (XZ) | ⭐⭐⭐⭐⭐ **极易** | 同上 |
| **山峰** | 2D (XZ) | ⭐⭐⭐⭐⭐ **极易** | 同上 |
| **温度** | 2D (XZ) | ⭐⭐⭐⭐⭐ **极易** | 同上 |
| **湿度** | 2D (XZ) | ⭐⭐⭐⭐⭐ **极易** | 同上 |
| **洞穴** | 3D (XYZ) | ⭐⭐⭐⭐ **容易** | 3D 噪声增加计算量，但每个体素仍独立，天然并行 |

**结论**：所有 7 个噪声层**从算法本质上看都极其适合 GPU 并行计算**，因为：
- 每个噪声采样点与其他采样点**完全独立**（无数据依赖）
- Simplex 噪声的数学运算（梯度向量点积、平滑插值）是 GPU 擅长的工作负载
- 2D 噪声的工作组尺寸可以设为 `(8, 8, 1)`，3D 可设为 `(4, 4, 4)`，利用率高

### 10.2 GPU 迁移的瓶颈：回读延迟

问题不在"能不能算"，而在"算完后怎么办"。

```
CPU 端调用 fill_terrain()
    │
    ▼
需要知道每个方块是什么 → 需要噪声采样结果
    │
    ▼
方案 A: GPU 算 → 写回 Staging Buffer → CPU 读回 → 填充 ChunkData → 提交网格
    │                                      ↑
    │                         此处有 GPU→CPU 回读延迟（通常 ~0.1-0.5ms）
    │                         使得 GPU 的加速优势被抵消
    ▼
方案 B: 全 GPU 管线 → GPU 算噪声 → GPU 填充方块 → GPU 生成网格 → 直接渲染
    │
    无需回读，但需要将调色板压缩、方块填充逻辑全部移植到 WGSL
```

### 10.3 三种 GPU 迁移方案对比

| 方案 | 说明 | 优点 | 缺点 | 推荐时机 |
|------|------|------|------|---------|
| **A：GPU 噪声 + CPU 回读** | GPU 计算所有噪声层，写回 Staging Buffer，CPU 读回后填充 ChunkData | 改造量小，保留现有调色板逻辑 | 回读延迟抵消加速收益；2D 场景甚至**更慢** | ❌ **不推荐**（性价比低） |
| **B：全 GPU 地形管线** | GPU 计算噪声 → 判定方块类型 → 输出 BlockId[] 到 Buffer | 消除回读；批量提交时吞吐量高 | 需要移植调色板压缩到 WGSL；复杂度高 | ✅ **推荐**（当 3D 洞穴成为瓶颈时） |
| **C：混合（推荐中期）** | 2D 噪声层在 CPU 算（开销低），仅 3D 洞穴噪声迁移到 GPU | 改造量最小；针对瓶颈精准优化 | 架构不统一 | ✅ **当前阶段推荐** |

### 10.4 性能对比估算

以每帧加载 4 个区块、每个区块 32³ 体素为基准：

| 场景 | CPU 纯算 | 方案 A | 方案 B | 方案 C |
|------|---------|--------|--------|--------|
| 仅 2D 噪声（6层） | ~0.12ms | ~0.3ms¹ | ~0.08ms | ~0.12ms |
| 2D + 3D 洞穴 | ~0.52ms | ~0.6ms¹ | ~0.15ms | ~0.22ms² |
| 多人 20 人（80 区块） | ~10.4ms | ~12ms¹ | ~3ms | ~4.4ms² |

> ¹ 回读延迟（~0.2ms/批）在小批量下占主导
> ² 仅 3D 洞穴噪声走 GPU

**关键 insight**：在当前单人或少量玩家场景下，CPU 端 7 层噪声的总耗时（~0.2ms/区块）远低于网格生成（~2-5ms/区块），**GPU 迁移的收益极低**。只有当 3D 洞穴 + 多人场景将噪声推到 10ms+ 级别时，GPU 迁移才真正有价值。

### 10.6 关于 [`docs/GPU噪声迁移方案.md`](../docs/GPU噪声迁移方案.md) 的补充

现有 GPU 噪声方案文档分析的是**单层 2D FBM**（当前实现），结论是"不必急于 GPU 化"。本文档的多噪声层方案**不影响这个结论**：

| 对比项 | 原有单层 FBM | 本方案 7 层噪声 |
|--------|-------------|----------------|
| 每区块 2D 采样量 | 1024 次 | 6144 次（6 层） |
| 每区块耗时 | ~0.01ms | ~0.06ms（2D 部分） |
| 是否成为瓶颈 | ❌ 否 | ❌ 否（仍远低于网格生成） |

> **GPU 迁移决策规则**：当 `噪声耗时 > 网格耗时 × 10%` 时启动迁移。当前比例约为 `0.2ms / 2ms = 10%`，处于临界值。如果未来网格生成优化（如 Greedy Meshing 将网格耗时降到 0.3ms），则噪声占比会上升到 40%，那时才是 GPU 迁移的真正触发点。

### 11.4 设计哲学：渐进式地貌丰富

不追求一次覆盖所有地貌，而是采用**可扩展的噪声层架构**：

```rust
/// 噪声层注册表：新增地貌只需添加新的噪声层
pub struct NoiseLayerRegistry {
    layers: Vec<Box<dyn NoiseLayer>>,
}

pub trait NoiseLayer {
    fn name(&self) -> &str;
    fn sample_2d(&self, x: f64, z: f64) -> f64;
    fn sample_3d(&self, x: f64, y: f64, z: f64) -> f64;
}
```

新增火山地貌示例（Phase 1 可加）：

```rust
// 新增火山热点噪声层
pub struct VolcanoNoise {
    hotspot: Fbm<Simplex>,   // 热点分布
    magma: Fbm<Simplex>,     // 岩浆房
}

impl NoiseLayer for VolcanoNoise {
    fn sample_2d(&self, x: f64, z: f64) -> f64 {
        self.hotspot.get([x * 0.001, z * 0.001]) // 超低频
    }
}
```

这种设计使得 Phase 0 实现核心 7 层后，后续扩展火山/冰川/喀斯特等地貌只需要添加新的 `NoiseLayer` 实现，**无需改动已有代码**。
