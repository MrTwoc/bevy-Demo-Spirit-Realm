## 核心架构（必读）

### [架构总纲.md](./架构总纲.md)
**项目整体架构蓝图**。整合了空间管理、LOD、剔除系统、渲染管线的完整方案。
- 目标规模：Y轴 ±20480、XZ无限、128+ 区块可视距离
- 整体架构：动态近景（0~16区块）+ 静态远景（17~128+区块）
- SubChunk 分层：32³（当前），HashMap 稀疏索引
- 渲染管线：Frustum Culling + Hi-Z Occlusion + MultiDrawIndirect

### [体素管理方案.md](./体素管理方案.md)
**极致性能体素引擎设计方案（GPU 加速全栈）**，77KB 详细设计文档。
- GPU Compute Shader 面提取（WGSL）
- SuperChunk 合批（8×8×4 区块合并为单一 Draw Call）
- MegaLOD 远景瓦片（64×64×16 区块，远景专用）
- ChunkData 三态存储：Empty / Uniform / Mixed（调色板压缩）
- 区块生命周期：Unloaded → Generating → Dormant → Active → Unloaded
- 邻居数据打包跨区块面剔除方案
- Bevy ECS 系统集成示例

## 地形生成

### [多噪声层地形生成方案.md](./多噪声层地形生成方案.md)
**七层噪声地形生成完整设计**，约 1183 行详细技术文档。
- 7 层噪声系统：大陆、高程、侵蚀、山峰、温度、湿度、洞穴（3D Simplex）
- 6 大地形类型：平原、丘陵、高原、山脉、盆地、海洋
- 垂直地层系统：支持 ±10240 Y 范围的地层分层
- 洞穴系统（3D Simplex 噪声）
- 河流系统 & 峡谷侵蚀算法
- 传说级地形设计（巨型森林等特殊生态）
- GPU Compute 可行性分析：Phase 0（CPU）→ Phase 3（GPU 迁移）

### [GPU噪声迁移方案.md](./GPU噪声迁移方案.md)
**WGSL Compute Shader 噪声迁移技术分析**，评估 GPU 加速的性价比。
- 关键发现：当前 CPU 2D 噪声非性能瓶颈（~0.01ms/chunk）
- 真实瓶颈：网格生成（~0.5-1.5ms/chunk）
- 三种迁移路径对比：A（GPU 噪声 + CPU 回读）、B（全 GPU 管线）、C（混合）
- 网格优化分析：Greedy Meshing、Compute Shader 面提取、SuperChunk 合批
- 结论：GPU 噪声优先级低于网格优化，建议将资源集中于网格管线
