# Spirit Realm 完整 Phase 规划

> **目标**：实现 128+ 区块视距，保持 60+ FPS，支持 Y=±20480 超大垂直范围。
>
> **最终目标**：在高配硬件（RTX 4070+）下达到 2048 区块视距 @144 FPS。
>
> **参考基准**：Voxy 模组在 GTX 1060 + i7-8700K + 16GB RAM 上，256 区块视距，~270 FPS，内存 ~3GB。

---

## Phase 2：GPU 面剔除 + MultiDrawIndirect — ❌ 待开始

> **对应文档**：[`Voxy借鉴优化方案.md §3.4`](Voxy借鉴优化方案.md:403)
> **对应文档**：[`架构总纲.md §5.1`](架构总纲.md:175)（GPU 视锥剔除 + Hi-Z 遮挡剔除）
> **依赖**：Phase 0 ✅
> **预估时间**：5-7 天

### 2.1 工作内容

| 工作项 | 难度 | 说明 |
|--------|------|------|
| **GPU 面剔除（Compute Shader）** | 🔴 高 | 将 CPU 逐面检查迁移到 WGSL Compute Shader |
| WGSL 计算着色器编写 | 🔴 高 | `@compute @workgroup_size(4,4,4)` 对体素并行面剔除 |
| ChunkData 上传到 GPU Buffer | 🟡 中 | 将区块体素数据 + 邻居数据封装为 GPU 可读格式 |
| 顶点数据回读或直接写入 GPU Mesh Buffer | 🔴 高 | 从 Compute Shader 输出顶点到 GPU 显存 |
| **MultiDrawIndirect** | 🟡 中 | 合并所有可见区块的 Draw Call 为 1 次提交 |
| 视锥剔除 + Hi-Z 遮挡剔除 | 🟡 中 | 在 Compute Shader 中剔除不可见区块 |
| Bindless 纹理支持 | 🟡 中 | 减少材质切换带来的 Draw Call 拆分 |

### 2.2 渐进式迁移策略

由于 Bevy 的 Compute Shader 支持仍在发展中，建议分阶段实施：

1. **阶段 A**：使用 `bevy::render::experimental::compute` API
2. **阶段 B**：如果 Bevy 支持不足，考虑使用 `wgpu` 直接调用
3. **阶段 C**：回退到 CPU 多线程方案（使用 Rayon）

### 2.3 预期收益

| 指标 | 当前 | 优化后 |
|------|------|--------|
| Draw Call | ~600-800 | **~100**（GPU 合批后） |
| CPU 面剔除耗时 | ~0.5ms/chunk | **<0.1ms/chunk** |
| CPU 提交开销 | ~2ms（Draw Call 批量提交） | **<0.5ms** |
| 渲染距离上限 | 8 区块（DC 瓶颈） | **32 区块**（DC 瓶颈解除） |

### 2.4 风险

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|----------|
| Bevy Compute Shader 支持不足 | 中 | 高 | 回退到 wgpu 直接调用或 CPU 多线程 |
| GPU 内存不足 | 低 | 高 | 动态调整 LOD 距离 |

## Phase 3：MegaLOD Tile 远景系统 — ❌ 待开始

> **对应文档**：[`架构总纲.md §3`](架构总纲.md:60)（区块分层设计）
> **对应文档**：[`LOD系统设计文档.md §8.2-8.3`](LOD系统设计文档.md:694)（MegaLOD Tile 定义）
> **依赖**：Phase 2（GPU 面剔除）
> **预估时间**：3-5 天

### 3.1 工作内容

| 工作项 | 说明 |
|--------|------|
| **MegaLOD Tile 定义** | 64×64×16 SubChunks 合并为一个远景瓦片 |
| 瓦片网格生成 | 对 64×64×16 区域做体素降采样 + 面剔除 |
| 瓦片序列化/磁盘缓存 | 预生成 Mesh 存储到磁盘，运行时直接加载 |
| 版本号驱动的增量更新 | 检测 SubChunk 变化，后台重烘焙受影响瓦片 |
| 瓦片 LOD 切换 | 33-64 区块 LOD1(1:16)，65-128 区块 LOD2(1:32) |
| 距离雾效掩盖 LOD 切换 | Bevy `FogSettings` Linear 雾 |

### 3.2 预期收益

| 指标 | 无 MegaLOD（32 区块） | 有 MegaLOD（128 区块） |
|------|---------------------|----------------------|
| 渲染实体数 | ~32,000 逐区块 | ~1,200 瓦片 + ~32,000 近景区块 |
| 总面数 | ~17.6M | ~28.4M |
| Draw Call | ~32,000 | ~1,200（近处逐区块 + 远处瓦片） |
| 渲染距离 | 32 区块 | **128+ 区块** |

### 3.3 与 Phase 2 的配合

MegaLOD Tile 的 Draw Call 通过 Phase 2 的 MultiDrawIndirect 合并：

```
近景（0-32 区块）：~32,000 个逐区块实体 → GPU MultiDraw → ~50 次调用
远景（33-128 区块）：~1,200 个 MegaLOD Tile → GPU MultiDraw → ~50 次调用
合计：~100 次 Draw Call
```

## Phase 4：多级空间索引 — ❌ 待开始

> **对应文档**：[`Voxy借鉴优化方案.md §3.6`](Voxy借鉴优化方案.md:476)
> **对应文档**：[`架构总纲.md §3.2`](架构总纲.md:100)（WorldColumn → MegaColumn → SubChunk）
> **预估时间**：2-3 天

### 4.1 工作内容

| 工作项 | 说明 |
|--------|------|
| `WorldColumn` 定义 | XZ 平面列坐标，含 LRU 淘汰 |
| `MegaColumn` 定义 | 垂直大列：`HashMap<i32, SubChunkEntry>` + Y 范围 |
| 列式加载/卸载 | 按列批量操作，而非逐区块 |
| LOD 列级生成 | 按列生成低精度地形高度图辅助剔除 |

### 4.2 预期收益

- O(1) 列查找 + O(1) SubChunk 查找
- 列级批量加载/卸载（减少逐区块开销）
- Y 轴流式加载优化（支持 Y=±20480）
