## [P1 内存泄漏修复 + 死代码清理 + 剔除验证] - 2026-05-06

### 新增

- **src/chunk_dirty.rs**：`ChunkMeshHandle` 组件，追踪每个 chunk 实体的 mesh 和 material handle

### 修改

- **src/chunk.rs**
  - `spawn_chunk_entity`：返回的实体现在包含 `ChunkMeshHandle` 组件，存储 mesh 和 material handle
  - 新增 `use crate::chunk_dirty::ChunkMeshHandle;`
  - 添加注释说明 Bevy 自动为 `Mesh3d` 实体计算 `Aabb`，启用视锥剔除和遮挡剔除

- **src/chunk_dirty.rs**
  - `rebuild_dirty_chunks`：重建 mesh 前先调用 `meshes.remove(old_mesh)` 和 `materials.remove(old_mat)` 释放旧资源
  - Query 参数从 `&Mesh3d` + `&MeshMaterial3d<StandardMaterial>` 改为 `Option<&ChunkMeshHandle>`
  - 重建后插入新的 `ChunkMeshHandle` 组件

### 剔除优化验证

- **视锥剔除（Frustum Culling）**：Bevy 自动为 `Mesh3d` 实体计算 `Aabb`，无需手动添加。当 `RENDER_DISTANCE` 增大时效果显著
- **遮挡剔除（Occlusion Culling）**：Bevy 内置支持，与 `Aabb` 配合自动工作
- **面剔除（Face Culling）**：已有，`is_face_visible()` + `ChunkNeighbors` 跨区块面剔除

### 删除

- **src/atlas.rs**：已废弃的硬编码 Atlas 槽位定义模块（项目已迁移到 `resource_pack.rs` 的动态 Atlas 系统）

### 修复

- GPU 内存泄漏：chunk mesh 重建时旧的 `Mesh` 和 `StandardMaterial` 资源现在会被正确从 `Assets` 中移除

## [Plan B + chunk_dirty] - 2026-05-05

### 新增

- **src/atlas.rs**：AtlasSlot 结构体、UV 计算、grass/dirt/stone 子模块 slot 定义
- **src/chunk.rs**：`BlockTexture` 枚举 + `from_block_and_face()` + `atlas_slot()`
- **src/chunk_dirty.rs**：`DirtyChunk` 组件、`mark_chunk_dirty`、`rebuild_dirty_chunks` 系统、`set_block_dirty`

### 修改

- **src/chunk.rs**
  - `generate_chunk_mesh`：移除顶点颜色，返回值从 5 元组改为 4 元组
  - `face_quad`：新增 `block_id` 参数，UV 改为 atlas 真实映射
  - `spawn_chunk_entity` / `spawn_initial_chunks`：材质改为 `base_color_texture`，加载 `textures/array_texture.png`
  - 移除：`block_color_as_rgba`、`face_uvs`、`ATTRIBUTE_COLOR`

- **src/chunk_dirty.rs**：同步更新为纹理方案

- **src/main.rs**：新增 `mod atlas;`、`mod chunk_dirty;`

### 纹理 atlas 规格

- 文件：`assets/textures/array_texture.png`，尺寸 250×1000px，子图 32×32px
- UV 槽位：grass_top → (0,0)，grass_side → (1,0)，dirt → (0,1)/(1,1)，stone → (2,0)

### 已知限制

- 旧 mesh/material 未显式从 `Assets` 移除，轻微 GPU 内存积累
- `ChunkMeshHandle` 组件未实现

## [初始化] - 2026-05-05

- 项目初始化，基于 Bevy 0.18.1

- 模块：main.rs、camera.rs、cube.rs、input.rs、chunk.rs、chunk_dirty.rs、chunk_wire_frame.rs、hud.rs
